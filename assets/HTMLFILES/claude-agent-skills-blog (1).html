01 What Are Claude Skills?

Every time you ask Claude to help with something, you have to re-explain your rules — "use TypeScript, write tests, follow our style guide..." That gets exhausting. Skills fix this. A Skill is a folder with a file called SKILL.md that tells Claude how to behave for a specific task — written once, reused forever. Think of it like a recipe card: give the chef the card once, and they never need instructions again.

💡 One-Line Definition

A Skill = a folder + a SKILL.md file. Write your rules once. Claude follows them every time.

02 How a Skill is Built

my-skill/ ├── SKILL.md ← Main instructions (required) ├── examples/ ← Good/bad output examples (optional) └── templates/ ← Reusable templates (optional)
--- name: code-review description: Reviews code for bugs, style, and security --- When reviewing code, check for: bugs, readability, security issues, and tests. Be kind — give a specific fix for every problem you find.
🧠 Progressive Disclosure

Claude only loads the extra files when needed — saving context (AI "brain space") for the actual work. Like a textbook appendix: only open it when you need it.

03 The 4 Types of Skills

🏢

Enterprise

Used across a whole company. IT admin sets it up once — 500 employees all get the same safe, consistent Claude behavior.

👤

Personal

Just for you. A student creates a my-notes Skill so Claude always formats notes their way — automatically.

📁

Project

Lives inside one project's folder. A StudyBuddy app Skill tells Claude: "React + Tailwind + Vitest, always." No re-explaining every session.

🔌

Plugin

Pre-packaged and shareable — like installing an app. Anthropic ships official ones (Excel, PowerPoint). Install once, get expert behavior instantly.

🏢 Enterprise Example

A hospital's "patient-safety" Skill: always recommend seeing a doctor, never give drug doses, use plain language. Every staff member's Claude follows this automatically.

🔌 Plugin Example

A marketing team installs an "SEO-writing" Skill — H2/H3 tags, short paragraphs, 8th-grade reading level. Expert SEO knowledge, zero training required.

06 Creating Custom Skills

You can build a Skill for anything. The course walks through two examples:

--- Practice Question Generator --- Given study notes: 1. Find the 5 key concepts 2. For each: write 1 multiple-choice + 1 short-answer question 3. Put answers at the bottom. Mix easy/medium/hard difficulty.

The second example is a Time Series Analysis Skill — tells Claude which stats to run, what charts to make, and how to explain results to non-technical readers.

✍️ Golden Rules for Writing Skills

Be specific. Show examples of good vs. bad output. Keep SKILL.md short. Test with different inputs. Iterate.

09 Tips & Tricks — Use Skills Like a Pro

🧪 Test Edge Cases

Try weird inputs — blank docs, wrong language, missing data. Fix gaps before relying on the Skill.

📦 One Job Per Skill

Don't cram everything in one Skill. Split "code review + security + tests" into three separate Skills.

🗣️ Write Like Talking to a New Intern

Be explicit: "We use React 18" not just "use the framework."

📎 Examples Beat Rules

Showing good vs. bad output is 10x more effective than an abstract rule.

🔢 Version Control Skills

Put Skills in Git. Update when your process changes. Roll back if something breaks.

🧩 Chain Skills Together

Data Skill → Presentation Skill → Email Skill = a full automated workflow.

🔍 Use the Skill Creator Skill

Ask Claude (with the meta-Skill loaded) to help you write new Skills. Extremely effective.

📏 Keep SKILL.md Short

Under 200 lines. Move details into subfolders — Claude loads them on demand.

🌐 Open Standard = Future-Proof

Skills work with any compatible agent, not just Claude today. Build once, use anywhere.

🔄 Review Monthly

Your process evolves. Update your Skills or they'll give outdated guidance.

10 Practice Labs — Do It Yourself

🧪 Lab 1 — Your First Personal Skill · Beginner · 30 min
1

Create a folder

Name it my-first-skill.

2

Write SKILL.md

Describe exactly how you want study notes formatted — bullet points, bold headers, summary at the end?

3

Test in Claude.ai

Paste your SKILL.md + some class notes. Did Claude follow your format?

4

Iterate

Try 3 different note sets. Fix whatever Claude got wrong. Repeat.

🧪 Lab 2 — Project Skill · Intermediate · 45 min
1

Pick a project

Any code project — even a simple HTML page works.

2

List the rules

Language, naming conventions, folder structure, common tasks.

3

Write SKILL.md

Include a front-matter header (--- block) with name and description.

4

Test it

Ask Claude to "add a dark mode toggle." Did it follow your conventions? Fix and repeat.

🧪 Lab 3 — Use a Pre-Built Skill · Any Level · 1 hr
1

Download the PowerPoint Skill

Find it on Anthropic's GitHub or docs page.

2

Load in Claude.ai

Add the Skill folder to your session.

3

Ask for a presentation

"Create a 5-slide deck on climate change for 10th graders."

4

Customize it

Add "include a fun fact on every slide" to the SKILL.md and re-run. Compare the results.

🧪 Lab 4 — Skill-Combo Workflow · Advanced · 2 hrs
1

Pick a multi-step task

Example: Research → Notes → Blog Post → Social Caption.

2

One Skill per step

Write research-notes, blog-writer, and social-caption Skills — each short and focused.

3

Chain them

Run Skill 1 → feed output into Skill 2 → feed into Skill 3. Notice the quality difference.

4

Share it

Write a README. Give it to a friend. If they can run the workflow without your help — it's a great Skill.
